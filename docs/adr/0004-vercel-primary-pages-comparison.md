# ADR-0004: Vercel 作为唯一主站，Pages 暂作对照站

网站需要保留两条已上线的技术路线，但两个正式品牌入口会分散内容维护、Canonical 和搜索信号。因此，GitHub + Vercel 版作为唯一主站；GitHub Pages 版作为公开可访问但不参与搜索竞争的对照站，用于验证纯静态路线，不承诺故障接管。

Pages 版只有在同时具备可验证的内容同步、明确的域名切换机制和约定的恢复目标后，才能升级为灾备站。在此之前，不使用“灾备站”描述它。ADR-0007 进一步确定了从对照站建设为温备站的目标架构，ADR-0012 确定了恢复与新鲜度目标；域名切换机制及其演练仍需另行决策。

## Consequences

- 品牌、Canonical 和搜索索引统一指向主站。
- 对照站可以继续公开比较静态实现，但应避免被搜索引擎作为另一份正式内容收录。
- 两版可以保留不同的渲染与部署实现；真实内容是否以及如何共享，另行决策。
- 如果未来升级为灾备站，需要补充切换流程、恢复验收和定期演练。

## Alternatives Considered

**两个同等正式网站**

- 未采用：会造成品牌入口、外链、搜索信号和内容维护分裂。

**立即将 Pages 定义为灾备站**

- 未采用：当前没有内容同步、域名切换和恢复目标，无法兑现故障接管承诺。
