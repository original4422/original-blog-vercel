# ADR-0010: 已发布版本零丢失与一小时人工恢复

Pages 温备站必须与主站最近一次成功发布保持相同内容，恢复点目标为零个已发布版本丢失；草稿和未完成发布不属于恢复承诺。Vercel 持续故障时采用人工确认和域名切换，恢复时间目标不超过一小时，不建设自动故障切换。

故障接管不得依赖临时修改源码或从头恢复内容。Pages 的最后一次已知良好版本必须已经完成构建、部署和基本访问验证，域名切换只负责把访问入口指向这个现成版本。

## Consequences

- 一个版本只有在主站与 Pages 温备所需产物都验证成功后，才能被视为正式发布。
- 发布流程需要避免主站先公开一个 Pages 尚未准备好的新版本；具体顺序由 ADR-0011 决定。
- 短暂 Vercel 故障可以等待恢复；确认持续故障后，由人工在一小时目标内执行切换。
- 不承担秒级恢复或无人值守自动切换的复杂度和误切风险。
- 需要独立记录最近一次已验证的主站部署、Pages 部署和共同源提交。

## Alternatives Considered

**允许 Pages 落后一个或多个正式版本**

- 未采用：故障接管会回退已经公开的内容，不符合零个已发布版本丢失的目标。

**自动健康检查和自动域名切换**

- 未采用：个人网站的业务影响不足以抵消误判、DNS 编排和持续运维成本。

**故障发生后临时构建 Pages**

- 未采用：恢复成功依赖构建链和源码平台当时可用，无法保证一小时恢复目标。
